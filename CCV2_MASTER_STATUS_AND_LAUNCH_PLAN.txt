3) IN-DEPTH GO-LIVE PLAN (END-TO-END)
==================================================

PHASE 0 — Freeze + branch discipline
1. Stop feature creep for launch window.
2. Keep fixes small and test after each change.
3. Ensure both branches are intentional:
   - staging = pre-prod validation
   - main = production release

PHASE 1 — Local launch hardening (must pass)
1. Start clean local dev:
   - kill stale Next processes
   - run dev on :3000
2. Type safety:
   - npm run tsc (must pass)
3. Route behavior checks:
   - / (must follow final desired behavior)
   - /dashboard
   - /tools/image-generation
   - /tools/video-generation
   - /tools/image-upscale
   - /tools/influencers
4. Auth gating checks:
   - signed-out user can access intended public pages only
   - protected pages force auth where intended
5. Core function tests (signed-in):
   - image generate (Nano Banana Pro + Nano Banana 2 + Edit)
   - video generate (Kling)
   - upscale (Topaz + premium models by plan)
   - character training end-to-end

PHASE 2 — Pricing, plans, and credit verification
1. Free / Starter / Creator / Studio access matrix re-test.
2. Ensure pricing text matches backend enforcement.
3. Confirm credit deduction variants are correct per tool/model.
4. Validate upgrade prompts and plan-required errors are clear.

PHASE 3 — GitHub release readiness
1. Commit only validated changes.
2. Push to staging.
3. Validate staging deploy + smoke test.
4. Merge/cherry-pick to main only after staging green.
5. Tag launch commit.

PHASE 4 — Railway staging + production configuration
For EACH environment (staging and production):
1. Service source branch correct:
   - staging env service -> staging branch
   - production env service -> main branch
2. PostgreSQL attached.
3. Required variables set:
   - DATABASE_URL
   - NODE_ENV=production
   - NEXT_PUBLIC_APP_URL
   - APP_URL
   - FAL_API_KEY
   - CLERK_SECRET_KEY
   - NEXT_PUBLIC_CLERK_PUBLISHABLE_KEY
   - billing keys + webhook secrets
4. Deploy succeeds.
5. Run migrations separately (not inside build):
   - npx prisma migrate deploy
6. Verify service health and logs.

PHASE 5 — Payment + webhook validation
1. Check payment plan IDs map to DB subscription tiers.
2. Validate checkout paths:
   - free -> starter
   - starter -> creator
   - creator -> studio
3. Validate webhook events update entitlements.
4. Validate cancellation/downgrade behavior.

PHASE 6 — Production smoke test (final gate)
Run with fresh test accounts:
1. Free user flow
2. Starter user flow
3. Creator user flow
4. Studio user flow
5. Character training flow
6. Error monitoring for 5xx and failed provider jobs

PHASE 7 — Launch
1. Confirm no active blockers.
2. Publish to production.
3. Monitor first 60–120 minutes:
   - auth errors
   - payment webhook failures
   - model job failures
   - credit mismatch reports
4. Keep rollback plan ready.
